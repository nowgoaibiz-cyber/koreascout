```tsx
'use client'

const baseClasses =
  'inline-flex items-center text-xs font-medium px-2.5 py-1 rounded-full'

const variantClasses = {
  default: 'bg-[#F2F1EE] text-[#6B6860]',
  success: 'bg-[#DCFCE7] text-[#16A34A]',
  warning: 'bg-[#FEF3C7] text-[#D97706]',
  danger: 'bg-[#FEE2E2] text-[#DC2626]',
  info: 'bg-[#DBEAFE] text-[#2563EB]',
  'tier-free': 'bg-[#F2F1EE] text-[#6B6860]',
  'tier-standard': 'bg-[#DBEAFE] text-[#2563EB]',
  'tier-alpha': 'bg-[#DCFCE7] text-[#16A34A]',
} as const

export type BadgeVariant = keyof typeof variantClasses

export interface BadgeProps extends React.HTMLAttributes<HTMLSpanElement> {
  variant?: BadgeVariant
  className?: string
  children?: React.ReactNode
}

export function Badge({
  variant = 'default',
  className = '',
  children,
  ...rest
}: BadgeProps) {
  const variantClass = variantClasses[variant]
  return (
    <span
      className={`${baseClasses} ${variantClass} ${className}`.trim()}
      {...rest}
    >
      {children}
    </span>
  )
}
```

### 7-3. 정렬 방향

| 항목 | 값 | 비고 |
|------|-----|------|

| reportIndex | `currentIndex = idList.indexOf(id)` | **이미 존재** (0-based) |
| Rule of 3 | — | 목록을 “최신 3건” 등으로 제한하는 로직은 **아직 없음** |

### 7-4. Rule of 3 구현 시 영향

- **prev/next:** `idList`가 “전체 주간 id”이므로, Rule of 3로 **목록을 3건으로 자르면** prev/next도 그 3건 안에서만 움직이게 됨. 즉, `idList`를 `.slice(-3)` 또는 상위 3건으로 제한하는 순간 네비게이션 범위가 3건으로 제한됨.
- **쿼리 변경 제안:** Rule of 3를 “해당 주에서 최신 3건만”으로 둘 경우, `weekReports` 쿼리를 `order("created_at", { ascending: false }).limit(3)` 등으로 바꾸고, `idList`를 그 결과로 구성하면 됨. 기존 `currentIndex` / `prevId` / `nextId` 로직은 그대로 사용 가능.
- **구현 준비도:** **Ready.** reportIndex·prev/next·idList 구조 있음. Rule of 3는 “idList 생성 전에 limit(3) + order 방향”만 추가하면 됨.
